📅 2025-11-29 • 🏷️ Course • ✍️ Teungku Zulkifli
#gns3-series

Masuk ke samudra (Routing). Siapkan Router c3725 kamu!

STAGE 2: The Border Control (Routing Dasar)

🧠 Logika Dasar: Kenapa Switch Gagal di Sini?

Di Stage 1, PC1 dan PC2 ada di “satu ruangan” (Network yang sama: 192.168.1.x). Switch cuma butuh teriak (Broadcast ARP) untuk nemuin orang.

Di Stage 2, kita akan pisahkan mereka ke dua gedung berbeda (Beda Network/Subnet).

  • Gedung A (Network 10): 192.168.10.x
  • Gedung B (Network 20): 192.168.20.x

Masalahnya: Teriakan ARP (Broadcast) itu TIDAK BISA nembus tembok gedung. Switch gak bisa meneruskan teriakan ke network lain. Solusinya: Kita butuh Router. Router adalah Satpam Pintu Gerbang. Kalau PC1 mau kirim ke Gedung B, dia gak perlu teriak nyari PC2. Dia cukup titip paketnya ke Satpam (Gateway), nanti Satpam yang urus cara nyebrangnya.

🛠️ Lab Time: Menghidupkan Sang Router

Langkah 1: Bangun Topologi

Buat project baru.

  1. Tarik Router c3725 ke tengah layar.
  2. Tarik 2 VPCS (PC1 dan PC2).
  3. Hubungkan kabel (Perhatikan port-nya!):
    • PC1 colok ke Router FastEthernet0/0 (kita sebut f0/0).
    • PC2 colok ke Router FastEthernet0/1 (kita sebut f0/1).
    • Catatan: Kalau slot f0/1 gak ada, klik kanan router -> configure -> slots -> tambah NM-1FE-TX di slot 1.
  4. Nyalakan semua (Start).

Langkah 2: Setting PC (Kenalan sama Konsep Gateway)

Ini bedanya sama Stage 1. PC harus dikasih tahu: “Kalau mau keluar gedung, lewat pintu mana?”. Pintu itu adalah IP Router.

PC 1 (Warga Gedung A):

  • IP: 192.168.10.2
  • Pintu Gerbang (Gateway): 192.168.10.1 (Ini nanti IP Router f0/0)
  • Command di VPCS: bash ip 192.168.10.2 192.168.10.1 24 (Format: ip [alamat_kita] [alamat_gateway] [subnet_mask_singkat])

PC 2 (Warga Gedung B):

  • IP: 192.168.20.2
  • Pintu Gerbang (Gateway): 192.168.20.1 (Ini nanti IP Router f0/1)
  • Command di VPCS: bash ip 192.168.20.2 192.168.20.1 24

Langkah 3: The Surgeon Mode (Bedah Router Cisco)

Sekarang masuk ke konsol Router c3725. Tampilannya akan beda, lebih serem tapi keren. Ini “bahasa” standar engineer seluruh dunia.

  1. Klik kanan Router -> Console.
  2. Kalau ditanya “Would you like to enter initial configuration dialog? [yes/no]:”, ketik no lalu Enter. (Kita mau setting manual, bukan dipandu wizard bawel).
  3. Tekan Enter sampai muncul Router>.

Ikuti ini:

Router> enable

(Masuk ke mode admin/root. Tandanya berubah jadi #)

Router# configure terminal

(Masuk ke mode konfigurasi global. Tandanya berubah jadi (config)#. Kita mau rombak settingan)

Setting Pintu Kiri (arah ke PC1):

Router(config)# interface fastEthernet 0/0
Router(config-if)# ip address 192.168.10.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# exit

(PENTING: no shutdown itu wajib. Secara default, port router Cisco itu MATI. Harus dibangunin manual)

Setting Pintu Kanan (arah ke PC2):

Router(config)# interface fastEthernet 0/1
Router(config-if)# ip address 192.168.20.1 255.255.255.0
Router(config-if)# no shutdown
Router(config-if)# end

Langkah 4: Verification (Saatnya Pembuktian)

Sekarang Router sudah punya dua kaki di dua dunia berbeda.

  1. Cek apakah Router sudah “bangun”:

    bash show ip interface brief

    Pastikan status f0/0 dan f0/1 adalah UP.

  2. Cek “Peta” Router:

    bash show ip route

    Kamu harus melihat kode C (Connected).

    • C 192.168.10.0 is directly connected…
    • C 192.168.20.0 is directly connected… (Artinya: Router tahu, “Oh, network 10 ada di tangan kiri, network 20 ada di tangan kanan”.)

  3. FINAL BOSS: Buka Console PC1, lalu ping ke PC2 (nyebrang router).

    bash ping 192.168.20.2

🕵️‍♂️ Analisis Expert (Biar gak cuma bisa setting)

Kalau ping berhasil, perhatikan ini:

  1. TTL (Time To Live): Kalau kamu ping sesama LAN (Stage 1), TTL biasanya 64 atau 128. Tapi kalau lewat Router, angka TTL akan berkurang 1.
    • Router itu “memungut pajak”. Setiap paket yang lewat router, umurnya dikurangi 1. Ini mekanisme supaya paket nyasar gak hidup selamanya di internet (Zombie Packet).
  2. Wireshark (Opsional tapi Seru):
    • Kalau kamu sadap di kabel PC1, kamu akan lihat PC1 kirim ARP tanya: “Who is 192.168.10.1 (Gateway)?”. Dia TIDAK tanya “Who is 192.168.20.2 (PC2)?”.
    • PC1 cuma peduli cara nyampai ke Pintu Gerbang. Masalah nyebrang ke PC2, itu urusan Router.

🚦 Misi Selesai?

Coba lakukan langkah di atas. Goalnya:

  1. PC1 bisa ping Gateway-nya sendiri.
  2. PC1 bisa ping Gateway seberang.
  3. PC1 bisa ping PC2.

Kalau sukses (muncul Reply), kamu resmi jadi Network Admin Level 2. Kamu sudah bisa menghubungkan dua dunia!

Next Stage! Kalau sukses, kita akan bikin kekacauan di Stage 3 (Dynamic Routing) dimana kita akan pakai 3 Router sekaligus!
